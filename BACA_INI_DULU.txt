MASALAH 1: Error "Duplicate entry BT001" ❌
───────────────────────────────────────────
Saat user coba buat jabatan di hosting, muncul error:
"SQLSTATE[23000]: Integrity constraint violation: 1062 
 Duplicate entry 'BT001' for key 'jabatans_kode_jabatan_unique'"

PENYEBAB:
- Unique constraint pada kode_jabatan tidak include user_id
- User A punya BT001 → User B coba buat BT001 → ERROR!

SOLUSI: ✅
- Update controller: Generate kode per user
- Update database: Unique constraint (kode_jabatan, user_id)
- Sekarang setiap user bisa punya BT001 sendiri tanpa conflict

───────────────────────────────────────────────────────────────────────────────

MASALAH 2: User Baru Dapat Data Salah ❌
────────────────────────────────────────
User baru hanya dapat:
- 11 COA (salah, tidak sesuai Jasuke)
- 4 Satuan (kurang banyak)

PENYEBAB:
- Listener masih pakai DefaultCoaSeeder (lama)
- Seharusnya pakai DefaultCoaSeederBaru (50 COA Jasuke)

SOLUSI: ✅
- Update listener: Pakai DefaultCoaSeederBaru
- User baru sekarang dapat:
  → 50 COA (format Jasuke: Jagung, Susu, Keju, Kemasan, dll)
  → 16 Satuan (lengkap: KLG, ONS, KG, ML, G, LTR, PTG, EKOR, dll)
