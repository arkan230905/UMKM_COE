PEMBAYARAN BEBAN BUG FIX - SUMMARY
================================================================================

MASALAH:
--------
Error saat menyimpan pembayaran beban:
"SQLSTATE[23000]: Integrity constraint violation: 1452 Cannot add or update 
a child row: a foreign key constraint fails"

ROOT CAUSE:
-----------
Database memiliki 2 tabel akun:
1. coas (legacy)
2. accounts (current)

Foreign key di pembayaran_beban mereferensi accounts table, tetapi controller
menggunakan Coa model yang menghasilkan ID dari coas table → mismatch!

SOLUSI:
-------
Gunakan Account model untuk semua foreign key ke accounts table.

FILES YANG DIPERBAIKI:
----------------------
1. app/Models/PembayaranBeban.php
   - coaBeban() → belongsTo(Account::class)
   - coaKas() → belongsTo(Account::class)

2. app/Models/PelunasanUtang.php
   - akunKas() → belongsTo(Account::class)
   - coaPelunasan() → belongsTo(Account::class)

3. app/Models/ReturKompensasi.php
   - akun() → belongsTo(Account::class)

4. app/Http/Controllers/Transaksi/PembayaranBebanController.php
   - store() method: Coa → Account
   - Added user_id filter untuk multi-tenant

5. app/Http/Controllers/ExpensePaymentController.php
   - store() method: Coa → Account
   - Added user_id filter untuk multi-tenant

6. app/Models/Penjualan.php
   - Added created() hook untuk journal creation
   - Set payment_status = 'paid' saat create
